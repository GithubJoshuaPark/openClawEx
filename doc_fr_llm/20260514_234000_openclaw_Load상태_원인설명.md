## 1. 질문 요지
`RunAtLoad`와 `KeepAlive`를 모두 `<false/>`로 설정하고 PC를 재부팅했음에도 불구하고, `listOpenclaw` (`launchctl list`) 조회 시 아래와 같이 서비스가 목록에 나타나는 원인에 대한 분석입니다.

## 2. 원인 분석: macOS `launchd`의 에이전트 로드 메커니즘

결론부터 말씀드리면, **프로세스는 실행되지 않았지만 설정 파일 자체가 시스템 레지스트리에 등록(Load)되는 것이 macOS의 기본 동작**이기 때문입니다.

1. **자동 등록 (Load)**
   - macOS는 사용자가 로그인하는 순간 `~/Library/LaunchAgents/` 폴더에 존재하는 모든 `.plist` 파일을 읽어들여 내부 서비스 목록에 일단 등록(Load)합니다.
   - 설정 파일이 해당 경로에 존재하는 이상, 이 등록 과정은 부팅/로그인 시점에 항상 자동으로 일어납니다.

2. **실행 방지 (PID `-`)**
   - 설정 파일 내에서 `RunAtLoad`와 `KeepAlive`를 모두 `false`로 설정했기 때문에, `launchd`는 서비스를 목록에 등록만 해둘 뿐 **실제 프로세스(PID)를 자동으로 구동(Start)하지는 않습니다.**
   - 따라서 출력 결과의 첫 번째 열(PID)이 `-` (실행 중인 프로세스 없음)로 표시되는 것입니다.

3. **`killOpenclaw` 실행 시 사라지는 이유**
   - Joshua님께서 설정하신 `killOpenclaw`의 실제 명령어는 `launchctl unload ...`입니다.
   - `unload`는 레지스트리에 등록되어 있던 서비스 항목 자체를 해제하여 지우는 명령이므로, 실행 후 `listOpenclaw`를 조회하면 목록에서 완전히 사라지게 됩니다.

## 3. 결론 및 향후 운영 가이드

- **현재 상태는 완벽히 의도된 정상 동작입니다.**
  - 부팅 시 백그라운드 프로세스가 구동되어 시스템 메모리나 CPU 자원을 소모하는 것을 성공적으로 차단한 상태입니다.
- **권장 운영 방식**:
  - 평소 부팅 직후에는 PID가 `-`인 대기(Load) 상태로 두시다가, 게이트웨이가 필요할 때 `runOpenclaw`를 입력하여 프로세스를 구동하시면 됩니다.
  - 만약 부팅 시 목록(Load 상태) 자체에 아예 나타나지 않게 하려면 `.plist` 파일을 `LaunchAgents` 폴더가 아닌 다른 임의의 경로에 보관해야 합니다. 하지만 현재 상태로도 자원 소모가 전혀 없으므로, 현재 구조를 유지하며 Alias 명령어를 활용하시는 것이 관리상 가장 편리합니다.
