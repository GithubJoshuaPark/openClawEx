# openClaw 자동 실행 위치 파악 및 수동 실행으로의 전환

**작성일시**: 2026년 05월 14일  
**요청자**: Joshua  

---

## 1. 개요 및 원인 분석
현재 로컬 PC(MacBook Pro) 부팅 후 OpenClaw가 자동으로 동작하는 원인은 macOS 사용자 서비스 관리자인 **LaunchAgent** 설정에 있습니다.
- **설정 파일 위치**: `~/Library/LaunchAgents/ai.openclaw.gateway.plist`
- **자동 실행 원인**: 파일 내 `<key>RunAtLoad</key>` 속성이 `<true/>`로 설정되어 있어, OS 부팅(로그인) 시 `launchd` 데몬이 자동으로 프로세스를 실행합니다.

## 2. 변경 및 조치 사항

### ① 부팅 시 자동 실행 해제
`ai.openclaw.gateway.plist` 파일에서 `RunAtLoad` 속성을 `<false/>`로 수정하여 부팅 시 자동으로 실행되지 않도록 조치했습니다.

```diff
  <key>Comment</key>
  <string>OpenClaw Gateway (v2026.4.8)</string>
  <key>RunAtLoad</key>
- <true/>
+ <false/>
  <key>KeepAlive</key>
```

### ② 수동 실행 명령어(`runOpenclaw`) 개선
`RunAtLoad`를 `false`로 설정하면 기존 `launchctl load` 명령어만으로는 서비스가 즉시 구동되지 않고 등록만 되는 특성이 있습니다. 
따라서 터미널에서 `runOpenclaw` 입력 시 즉시 실행되도록 `~/.zshrc` 파일의 alias 설정을 아래와 같이 보완했습니다.

```diff
# OpenClaw 서비스 관리 (LaunchAgent)
  alias killOpenclaw="launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist"
  alias listOpenclaw="launchctl list | grep openclaw"
- alias runOpenclaw="launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist"
+ alias runOpenclaw="launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist && launchctl start ai.openclaw.gateway"
```

---

## 3. 향후 사용 안내
- **현재 구동 중인 서비스 끄기**: 터미널에서 `killOpenclaw`를 입력합니다.
- **필요 시 수동 실행하기**: 터미널에서 `runOpenclaw`를 입력합니다.
- 변경된 alias 설정을 현재 열려 있는 터미널 창에 즉시 적용하려면 `source ~/.zshrc`를 한 번 실행해 주시기 바랍니다.
